base tablesRead topic

What it is

base tables is a ServiceNow administration/development concept that affects how records, users, automation, security, reporting, and support processes behave in a real instance.

How it is used

Use it to design supportable configuration, troubleshoot behavior, document platform impact, and explain decisions to admins, developers, process owners, and auditors.

Where to look

Start from the relevant application menu, then inspect the table, record metadata, related lists, update records, security rules, logs, and official release-specific documentation.

Related tables

sys_db_objectsys_dictionarytask

Example use case

A platform owner receives a request to change base tables. A job-ready admin documents the requirement, table impact, security impact, test evidence, rollback plan, and support handover before moving it forward.

Common problems

  • Changing configuration without knowing the affected table or application scope.
  • Testing only as admin instead of impersonating real users.
  • Forgetting reports, integrations, mobile, workspace, and API behavior.

Practice lab

Build a small example in a PDI or sandbox. Capture screenshots, table names, configuration records, test users, and before/after behavior.

Job readiness evidence

Be able to explain the business requirement, platform mechanism, configuration path, security model, testing proof, and rollback plan in an interview or production review.
