How to study any topic

  1. Start with the definition and the business reason it exists.
  2. Look at where it is used in the platform and which table, script, or process owns it.
  3. Inspect a real example, then open the related record or configuration item.
  4. Check the permissions, runtime rules, and logs around it.
  5. Use the practice lab to prove you understand the behavior.

Official docs reference: use the ServiceNow documentation portal and search for the topic name.

Access and security

ACL

What it is: the rule that decides who can read, write, create, delete, or execute.

How it is used: secure tables, fields, scripts, and APIs.

Where to look: sys_security_acl, Security Debugger, impersonation, and the field or table being blocked.

Examples: table ACLs, field ACLs, script ACLs.

Common problems: admin can see it, end users cannot; a field is locked because a field ACL blocks it; inherited ACLs hide the real rule.

Practice lab: impersonate a non-admin user, open incident, and trace why the caller or assignment group field is hidden or read-only.

Docs: Official docs portal and search for ACL and contextual security.

Automation logic

Business Rule

What it is: server-side logic that runs before or after database actions.

How it is used: update related records, enforce rules, set default values, and log business decisions.

Where to look: the table's Business Rules related list, sys_script, and the system logs when the rule executes.

Examples: state changes, approvals, calculated fields, audit messages.

Common problems: recursion, slow scripts, using the wrong timing, or updating records in a loop.

Practice lab: add a before rule to incident that blocks closure without a close code, then verify the message in the form.

Docs: search the docs portal for Business Rules and server-side scripting.

Reusable code

Script Include

What it is: reusable server-side JavaScript.

How it is used: shared logic for GlideAjax, business rules, flows, and validation helpers.

Where to look: sys_script_include, the app scope, and whether the include is client-callable.

Examples: validators, lookups, API wrappers, helper methods.

Common problems: scope mismatch, missing client-callable flag, bad inheritance, or calling server-only code from the browser.

Practice lab: create a Script Include that returns the current user's manager and call it from a simple GlideAjax client script.

Docs: search the docs portal for Script Include, GlideAjax, and scoped application scripting.

Automation builder

Flow

What it is: declarative automation for tasks and approvals.

How it is used: trigger work from records, approvals, notifications, and integrations without writing the whole process in script.

Where to look: Flow Designer, flow execution details, and the record that triggered the flow.

Examples: incident routing, request fulfillment, approval chains, notification steps.

Common problems: trigger conditions that never fire, missing permissions, duplicated actions, and unclear rollback behavior.

Practice lab: build a flow on incident that assigns the record to a support group when priority is 1.

Docs: search the docs portal for Flow Designer and flow execution.

Configuration data

CMDB CI

What it is: a configuration item record that represents an asset, service, or technical component.

How it is used: service maps, dependency chains, impact analysis, governance, and discovery reconciliation.

Where to look: cmdb_ci, CI class hierarchy, relationships, and the IRE or discovery results.

Examples: servers, apps, databases, services, business services.

Common problems: bad identification, duplicate CIs, weak class hierarchy, and relationship noise.

Practice lab: open a CI record, trace its parent class, and identify the relationship used by a dependent service.

Docs: search the docs portal for CMDB, CSDM, and Identification and Reconciliation Engine.

Service operations

Incident

What it is: a record for restoring service quickly after an interruption or degradation.

How it is used: service desk intake, assignment, SLA tracking, communication, and closure.

Where to look: incident, assignment rules, SLA definitions, and the state model on the form.

Examples: broken email, VPN outage, application failure.

Common problems: poor categorization, missing assignment, weak closure codes, and untracked communications.

Practice lab: create a high-priority incident, move it through assignment, SLA update, and closure, then review the work notes.

Docs: search the docs portal for Incident management and service desk workflows.

Controlled change

Change Request

What it is: a controlled record for planned service change.

How it is used: approvals, scheduling, risk assessment, implementation, and review.

Where to look: change_request, change models, CAB approvals, and the change calendar.

Examples: server patching, app deployment, firewall change.

Common problems: weak risk data, missing backout plan, and bad scheduling windows.

Practice lab: open a normal change, add risk, route it for approval, and check how the task list reflects the lifecycle.

Docs: search the docs portal for Change management, approvals, and change calendar.

Self-service

Catalog Item

What it is: a requestable product or service in the service catalog.

How it is used: self-service requests, fulfillment, approvals, and orchestration.

Where to look: sc_cat_item, variables, catalog tasks, and the flow or workflow that fulfills the item.

Examples: laptop request, software request, access request.

Common problems: bad form design, unclear fulfillment mapping, missing approvals, and poor variable naming.

Practice lab: create a single-variable catalog item and route it to a task so you can see the end-to-end request flow.

Docs: search the docs portal for Service Catalog, catalog items, and variables.

System to system

Integration

What it is: data exchange between ServiceNow and another system.

How it is used: REST, SOAP, IntegrationHub, webhooks, imports, and outbound notifications.

Where to look: REST Message records, import sets, ECC Queue, integration logs, and the script or flow that sends the payload.

Examples: HR feed, CMDB sync, monitoring events, ticket sync.

Common problems: authentication, payload mapping, retries, bad transformations, and missing error handling.

Practice lab: trace one outbound REST call from its script/flow to its response log and identify the payload fields.

Docs: search the docs portal for REST API, IntegrationHub, and import set processing.

Support knowledge

Knowledge Article

What it is: documented solution or reference content for users and agents.

How it is used: deflection, self-service, agent assist, and support consistency.

Where to look: kb_knowledge, the knowledge base, article workflow, and visibility rules.

Examples: reset password article, incident troubleshooting guide.

Common problems: outdated content, poor taxonomy, and visibility issues.

Practice lab: publish a short article, search for it as a user, and verify that the right role can see it.

Docs: search the docs portal for Knowledge management and article workflow.

On-prem bridge

MID Server

What it is: the bridge between ServiceNow and local network resources.

How it is used: discovery, orchestration, integrations, and secure on-prem communication.

Where to look: MID Server status, ecc_queue, discovery logs, credentials, and network connectivity.

Examples: discovery probes, database connections, credentialed access.

Common problems: firewall rules, certificates, credentials, queued messages, and agent health.

Practice lab: check one ECC Queue item and trace it to the MID Server that should process it.

Docs: search the docs portal for MID Server, ECC Queue, and discovery troubleshooting.
