How to study any topic

  1. Start with the definition.
  2. Look at where it is used in the platform.
  3. Inspect a real example and the related table or script.
  4. Check the permissions, ACLs, or runtime rules around it.
  5. Learn the common failures and the first troubleshooting step.

Access and security

ACL

What it is: the rule that decides who can read, write, create, delete, or execute.

How it is used: secure tables, fields, scripts, and APIs.

Examples: table ACLs, field ACLs, script ACLs.

Common problems: admin sees it, but end users do not; a field is locked because a field ACL blocks it.

Automation logic

Business Rule

What it is: server-side logic that runs before or after database actions.

How it is used: update related records, enforce rules, and prepare values.

Examples: state changes, approvals, calculated fields.

Common problems: recursion, slow scripts, and unexpected updates.

Script Include

What it is: reusable server-side JavaScript.

How it is used: shared logic for GlideAjax, business rules, and flows.

Examples: validation helpers, API wrappers, lookup utilities.

Common problems: scope mismatch, missing client-callable flag, or bad inheritance.

Client Script

What it is: browser-side script that responds to form events.

How it is used: set defaults, show messages, react to field changes.

Examples: onLoad, onChange, onSubmit patterns.

Common problems: timing issues, unsupported APIs, and duplicate logic with UI Policies.

Flow

What it is: declarative automation for tasks and approvals.

How it is used: trigger work from records, approvals, and integrations.

Examples: incident routing, request fulfillment, notifications.

Common problems: trigger conditions, permissions, and duplicate actions.

CMDB CI

What it is: a configuration item record that represents an asset, service, or technical component.

How it is used: service maps, dependency chains, impact analysis, and governance.

Examples: servers, apps, databases, services, business services.

Common problems: bad identification, duplicate CIs, weak class hierarchy.

Incident

What it is: a record for restoring service quickly after an interruption or degradation.

How it is used: service desk intake, assignment, SLA tracking, communication, closure.

Examples: broken email, VPN outage, application failure.

Common problems: poor categorization, missing assignment, weak closure codes.

Change Request

What it is: a controlled record for planned service change.

How it is used: approvals, scheduling, risk assessment, implementation, and review.

Examples: server patching, app deployment, firewall change.

Common problems: weak risk data, missing backout plan, or poor scheduling.

Catalog Item

What it is: a requestable product or service in the service catalog.

How it is used: self-service requests, fulfillment, approvals, orchestration.

Examples: laptop request, software request, access request.

Common problems: bad form design, unclear fulfillment mapping, missing approvals.

Integration

What it is: data exchange between ServiceNow and another system.

How it is used: REST, SOAP, IntegrationHub, webhooks, imports, outbound notifications.

Examples: HR feed, CMDB sync, monitoring events, ticket sync.

Common problems: authentication, payload mapping, retries, and error handling.

Knowledge Article

What it is: documented solution or reference content for users and agents.

How it is used: deflection, self-service, agent assist, and support consistency.

Examples: reset password article, incident troubleshooting guide.

Common problems: outdated content, poor taxonomy, and visibility issues.

MID Server

What it is: the bridge between ServiceNow and local network resources.

How it is used: discovery, orchestration, integrations, and secure on-prem communication.

Examples: discovery probes, database connections, credentialed access.

Common problems: firewall rules, certificates, credentials, and agent health.
