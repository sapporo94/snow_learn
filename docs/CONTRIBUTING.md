# Contributing

This project should be edited like product documentation.

## Content rule

Every topic entry should include:

1. What it is
2. How it is used
3. Where to configure or inspect it
4. Related tables
5. Example use case
6. Common problems
7. Lab or practice task
8. Job-readiness evidence
9. References

## Branch naming

- `content/admin-tables-deep-dive`
- `feature/search-index`
- `fix/mobile-nav`

## Commit style

- `content: expand admin tables entry`
- `ui: improve module cards`
- `docs: add GitHub Pages deployment notes`
